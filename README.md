# ❄️ Healthcare Data Warehouse — Snowflake + Python + Power BI

> **End-to-End Cloud Data Warehousing Project**  
> Loading, transforming, and analysing 55,000+ hospital patient records using Snowflake, SQL analytics views, and Power BI dashboards.

---

## 📌 Project Overview

This project builds a cloud data warehouse on **Snowflake** for a healthcare dataset containing 55,000+ patient records. It demonstrates the full data engineering workflow — from raw CSV ingestion through a structured analytics layer to an interactive Power BI dashboard.

**Key Achievement:** Identified Obesity + Blue Cross as the highest-revenue segment and detected negative billing anomalies that would have caused downstream reporting errors.

---

## 🏗️ Architecture

```
Raw CSV (55k+ records)
        │
        ▼
┌─────────────────────────┐
│  Snowflake — RAW Schema │  ← PATIENTS table loaded via internal Stage
│  (Bronze Layer)         │
└─────────────────────────┘
        │
        ▼
┌──────────────────────────────┐
│  Snowflake — ANALYTICS Schema│  ← SQL Views (Window Functions, CTEs)
│  (Gold Layer)                │     Revenue, Admissions, Anomalies,
│                              │     Insurance, Trends, Doctor stats
└──────────────────────────────┘
        │
        ▼
┌─────────────────────────┐
│  Python Pipeline        │  ← snowflake-connector-python
│  snowflake_pipeline.py  │     Queries views → exports CSVs
└─────────────────────────┘
        │
        ▼
┌─────────────────────────┐
│  Power BI Dashboard     │  ← KPI cards, slicers, revenue charts
└─────────────────────────┘
```

## 🚀 How to Run

### Prerequisites
- Snowflake free trial account → [app.snowflake.com](https://app.snowflake.com)
- Python 3.8+
- Power BI Desktop

### Step 1 — Clone the repo
```bash
git clone https://github.com/Rohan9284/snowflake-healthcare-dwh.git
cd snowflake-healthcare-dwh
```

### Step 2 — Install Python dependencies
```bash
pip install -r requirements.txt
```

### Step 3 — Set up credentials
```bash
cp .env.example .env
# Edit .env with your Snowflake account details
```

### Step 4 — Run SQL setup in Snowflake worksheet
```
sql/01_setup.sql   → Run first (creates DB, schemas, warehouse)
sql/02_load_data.sql → Upload CSV to stage, then load
sql/03_analytics_views.sql → Creates all analytics views
```

### Step 5 — Run Python pipeline
```bash
python python/snowflake_pipeline.py
```
This queries all views and exports CSVs to the `exports/` folder.

### Step 6 — Open Power BI
- Open `powerbi/healthcare_snowflake.pbix`
- Or connect fresh: Get Data → Snowflake → enter account + warehouse
- Load the analytics views directly

## 📈 Key Business Insights

### 1. Top Revenue Segment
**Obesity + Blue Cross** is the highest-revenue combination — identified using SQL RANK() Window Functions across condition × insurance pairs.

### 2. Admission Type Breakdown
**33% of all admissions are Urgent** — indicating high ER resource demand. Elective admissions generate higher average billing per patient.

### 3. Billing Anomaly Detection
Detected **negative billing records** in the raw data that would have corrupted downstream revenue reporting — flagged via `vw_billing_anomalies` view.

### 4. Length of Stay vs Revenue
Urgent admissions have longer average stays but not proportionally higher billing — suggesting operational inefficiency worth investigating.

## 🔑 Snowflake Concepts Demonstrated

- **Virtual Warehouse** — compute engine (X-SMALL, auto-suspend)
- **Internal Stage** — uploading local CSV files for bulk loading
- **COPY INTO** — bulk data loading from stage to table
- **Schemas as layers** — RAW (bronze) and ANALYTICS (gold)
- **SQL Views** — reusable analytics queries on top of raw tables
- **Window Functions** — RANK(), SUM() OVER(), COUNT() OVER()
- **SnowSQL / Python Connector** — programmatic access to Snowflake
